### **(1) 뮤지컬 팬들을 위한 "뮤지컬 큐레이션 노트”**

#### 1. 기존 문제점

- 뮤지컬 관련 정보(공연 일정, 배우 활동, 리뷰 등)가 여러 사이트에 분산되어 있어 관리하기 어려움.
- 좋아하는 배우나 작품의 최신 소식을 각각 찾아보기 어려움

#### 2. 타겟 사용자

- ***뮤지컬 팬***:
    - 좋아하는 배우와 작품 정보를 정리하고 최신 소식을 받고 싶은 사람.
    - 공연 관람 기록을 저장하고 다른 팬들과 교류하고 싶은 사람.
- ***뮤지컬 입문자***:
    - 뮤지컬에 대한 정보를 쉽게 탐색하고 입문하고 싶은 사람.

#### 3. 주요 기능

1. **RSS/GCE 기반 콘텐츠 스크랩**:
    - 배우, 작품, 공연 관련 뉴스, 리뷰, 영상을 자동으로 스크랩.
    - 예: "조승우", "레 미제라블" 키워드로 콘텐츠 수집.
2. **개인화된 뮤지컬 캘린더**:
    - 좋아하는 배우나 작품 관련 공연 일정이 자동으로 추가되고, 알림 기능 제공.
3. **관람 기록 및 대시보드**:
    - 관람한 공연 기록 저장 및 통계 시각화.
    - "내가 가장 많이 본 배우", "이번 달 관람 횟수" 등 대시보드 제공.
4. **커뮤니티 및 소셜 기능**:
    - 팬 그룹 생성 및 작품별 토론 게시판 제공.
    - 다른 사용자가 추천한 콘텐츠를 확인하고, 댓글과 좋아요로 소통.

### **(2) 방 정리 마니아를 위한 미니멀리스트 정리 노트**

#### 1. 기존 문제점

- 미니멀리스트들은 물건의 필요성과 소유 여부를 체계적으로 기록하거나 관리할 도구가 부족함.
- 정리와 수납에 대한 조언을 주고받거나 정보를 공유할 플랫폼이 제한적임.

#### 2. 타겟 사용자

- **미니멀리스트**:
    - 물건을 최소화하고 깔끔한 공간을 유지하고 싶은 사람.
- **정리 컨설턴트**:
    - 정리와 수납 방법을 전문적으로 제공하거나 팁을 공유하고 싶은 사람.

#### 3. 주요 기능

1. **소유물 관리**:
    - 사용자가 현재 소유 중인 물건 목록을 기록하고 관리.
    - "3개월간 사용하지 않은 물건"을 자동으로 추천하여 비우기를 유도.
2. **정리 팁 큐레이션**:
    - RSS/GCE를 활용해 정리 및 수납 관련 블로그, 뉴스, 유튜브 콘텐츠를 자동 큐레이션하여 제공.
3. **정리 챌린지**:
    - "이번 주에 5개의 물건 비우기" 같은 개인 챌린지를 설정하고 결과를 기록.
    - 사용자가 챌린지 완료 시 뱃지 및 통계기록 제공.
4. **커뮤니티 연결**:
    - 사용자가 "사용하지 않는 물건"을 서로 교환하거나 중고 거래
    - 정리 노하우를 공유하거나 추천 콘텐츠를 피드에 게시.

### **(3) 현지인 맛집 추천**

타지에 가면 현지인이 추천하는 맛집을 가고 싶음.

당근마켓처럼 위치 인증을 해 현지인임을 인증(거주 기간 or 몇주간 현지인 인증) 본인의 맛집을 스크랩(네이버or카카오 매장 정보 등) 다른 사용자가 볼 수 있음. → 좋아요 기능 추가해 사람들이 더 많이 인정하는 맛집임을 확인할 수 있음.

#### 1. **기존 문제점**

- 여행이나 새로운 지역에 방문했을 때 믿을 만한 맛집 정보를 찾기 어렵고, 실제 현지인에게 식당을 추천받고 싶음.
- 기존 리뷰 플랫폼은 광고성 리뷰와 믿기 어려운 평가로 신뢰도가 낮음.

#### 2. **타겟 사용자**

- **여행자**:
    - 새로운 지역에서 진짜 현지인의 추천 맛집을 찾고 싶은 사람.
- **현지인**:
    - 자신의 맛집을 공유하고 추천받고 싶은 사람.

#### 3. **주요 기능**

- **현지인 인증(+거주 기간)**:
    - 위치 인증 및 거주 기간 정보를 기반으로 현지인을 인증.
    - 거주기간을 함께 표시해 맛집 신뢰도 상승
- **맛집 스크랩 및 리뷰**:
    - 현지인이 추천하는 맛집 정보를 스크랩하고, 신뢰도 높은 리뷰를 제공.
- **좋아요 기반 순위**:
    - 다른 사용자들의 "좋아요"로 인기 맛집 순위를 자동으로 결정.
- **관심 지역 설정**:
    - 사용자가 관심 있는 지역을 미리 설정해 관련 맛집 정보를 지속적으로 업데이트.

### (4) 고지서 한번에 정리하는 SNS

전기세, 수도세, 가스비 등 고지서가 각각 오고 납기일도 제각각이라 연체되는 경우가 많음.

과거에 얼마가 나왔는지도 기억이 잘 안나니 비교가 어려움.

⇒ 고지서가 오면 사진을 찍어 인식. 금액, 계좌번호, 납부기한 등을 저장해 자동으로 내 피드에 기록 + 알림 기능.

⇒ 같은 세대, 건물에 사는 사람들 요금도 확인하고 평균치를 알려주어 내 집 요금이 많이 나왔는지 가늠하는 기능도 추가 

#### 1. 기존 문제점

- 전기세, 수도세, 가스비 등 고지서가 각각 오고 납기일도 달라 연체 위험이 있음.
- 과거 고지서 기록을 비교하거나 평균 요금을 파악하기 어려움.

#### 2. 타겟 사용자

- **일반 가정 사용자**:
    - 다양한 고지서를 체계적으로 관리하고 싶은 사람.
- **임대인 및 공동 거주자**:
    - 동일 건물 또는 지역의 요금을 비교하고 싶어 하는 사용자.

#### 3. 주요 기능

- **고지서 데이터 기록**:
    - 고지서 사진을 찍으면 OCR로 납부 금액, 기한 등을 자동으로 기록.
- **알림 기능**:
    - 납부 기한이 다가오면 알림을 보내 연체를 방지.
- **커뮤니티 비교**:
    - 같은 지역 사용자의 평균 요금 데이터를 제공하여 본인의 요금과 비교
- **과거와 비교**
    - 월별 요금 변동 추이와 각 고지서의 비중을 한눈에 확인해 본인의 에너지 사용량을 시각화 → 환경보호

### 1. JPA 소개
- **JPA란?**
  - Java 진영의 ORM(Object-Relational Mapping) 기술 표준.
  - 객체와 관계형 데이터베이스 간의 패러다임 불일치를 해결.
  - Hibernate, EclipseLink 등 구현체를 통해 사용.

- **주요 특징**
  - 생산성: SQL 없이 객체 중심 개발 가능.
  - 유지보수성: 필드 변경 시 JPA가 SQL 자동 처리.
  - 성능 최적화: 1차 캐시, 쓰기 지연, 지연 로딩 지원.

- **장점**
  - 객체 지향 설계 반영 가능.
  - DB 벤더 독립적.
  - 표준 API 제공.

---

### 2. JPA 시작
- **프로젝트 설정**
  - H2 데이터베이스 및 Maven 사용.
  - JPA 설정 파일 `persistence.xml`을 통해 데이터베이스 연결 및 매핑 설정.

- **객체와 테이블 매핑**
  - `@Entity`, `@Table`을 사용해 클래스와 테이블 연결.
  - `@Id`로 기본 키 지정.

- **JPQL(Java Persistence Query Language)**
  - 객체를 대상으로 데이터를 검색하는 쿼리 언어.
  - SQL과 유사한 문법 제공.

---

### 3. 영속성 관리
- **영속성 컨텍스트**
  - 엔티티를 영구적으로 저장하고 관리하는 환경.
  - 1차 캐시, 동일성 보장, 쓰기 지연, 변경 감지, 지연 로딩 등 다양한 기능 제공.

- **엔티티 생명주기**
  1. 비영속: 영속성 컨텍스트와 관계없는 상태.
  2. 영속: 영속성 컨텍스트에 저장된 상태.
  3. 준영속: 영속성 컨텍스트에서 분리된 상태.
  4. 삭제: 엔티티가 삭제된 상태.

- **플러시(Flush)**
  - 영속성 컨텍스트의 변경 내용을 DB에 동기화.
  - 트랜잭션 커밋, JPQL 실행 시 자동 호출.

---

### 4. 엔티티 매핑
- **기본 매핑**
  - `@Entity`: 클래스와 테이블 매핑.
  - `@Column`: 필드와 컬럼 매핑.
  - `@Id`와 `@GeneratedValue`: 기본 키와 키 생성 전략 지정.

- **데이터베이스 스키마 자동 생성**
  - 개발 환경에서 DDL 자동 생성 (`create`, `update` 등).
  - 운영 환경에서는 `validate` 또는 `none` 권장.

- **기타 매핑 어노테이션**
  - `@Enumerated`: Enum 타입 매핑.
  - `@Lob`: 대용량 데이터(BLOB, CLOB) 매핑.
  - `@Transient`: 매핑에서 제외.

---

### 5. 연관관계 매핑 기초
- **연관관계 매핑의 핵심**
  - 객체는 참조를, 테이블은 외래 키를 통해 연관관계 설정.
  - 연관관계의 주인 설정이 중요 (외래 키 관리 주체).

- **단방향 vs 양방향**
  - 단방향: 한쪽에서만 참조 가능.
  - 양방향: 양쪽에서 참조 가능. 주인은 외래 키를 관리하고, 비주인은 조회만 가능.

- **매핑 예제**
  - 다대일: `@ManyToOne`
  - 일대다: `@OneToMany`
  - 일대일: `@OneToOne`
  - 다대다: `@ManyToMany` (실무에서 사용 비권장).
